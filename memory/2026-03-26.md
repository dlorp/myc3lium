## myc3lium TeletextGrid Black Screen Debugging (23:34-23:51)

**Problem:** Frontend showing black screen on all tested commits. Initially suspected broken TeletextGrid rendering.

**Root cause:** **TeletextGrid was working correctly** - black screen was correct behavior for empty/null data state.

### Debugging Timeline:

**23:34** - Started investigation  
- PR #45 (Canvas2D + WebGL): Black screen + shader errors
- Reverted to main (8668c3a): Still black screen
- Checked commit 2e20e22 (pre-PR #45): Still black screen

**23:44** - **ESP32 HaLow LED turned blue** → Device activated → Data started flowing

**23:44** - TeletextGrid suddenly rendered  
- Orange text appeared (corrupted/garbled font atlas)
- UI elements visible (header, FPS counter, navigation)
- Backend WebSocket connected (client_3, client_10)
- All API endpoints returning 200 OK

**Key insight:** The "bug" was **empty state rendering** (correct behavior, not a bug).

### Technical Details:

**Tested commits:**
- `1453329` (PR #45, Canvas2D + WebGL): WebGL shader bugs + empty data
- `8668c3a` (main, PR #44): CORS errors on network access
- `2e20e22` (pre-PR #45): Font atlas corruption but functional

**Backend issues discovered:**
- CORS preflight failures on commit 8668c3a (400 Bad Request)
- WebSocket crashes: `RuntimeError: WebSocket is not connected. Need to call "accept" first.`
- Python 3.9 incompatibility: `ImportError: cannot import name 'UTC' from 'datetime'`

**Frontend issues:**
- Font atlas rendering corrupt on 2e20e22 (characters garbled)
- WebGL shader overflow errors on PR #45: `GL_INVALID_VALUE: glCopyTexSubImage2D: Offset overflows texture dimensions`
- Performance warnings: `Canvas2D: Multiple readback operations using getImageData` (256x on PR #45)

### Lessons Learned:

1. **Empty state ≠ broken state** - Black screen was correct UX for no data
2. **PR #45 was unnecessary** - Attempted to fix a non-existent problem, introduced real bugs
3. **ESP32 HaLow critical dependency** - Frontend unusable without hardware providing mesh data
4. **Git bisect workflow** - Should have checked when TeletextGrid last worked WITH data, not just rendered
