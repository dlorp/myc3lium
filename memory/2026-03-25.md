### Key Findings

#### NES CHR ROM Graphics
- 8×8 pixel tiles, 2-bit color indices (4 colors via palette)
- Bit-plane separated storage (optimization for hardware decoding)
- 8KB CHR ROM = 512 tiles split into 2 banks
- **Critical insight:** Tiles store SHAPE only, COLOR via palette at render time
- Enables cheap animation via palette cycling

**Tool opportunities:**
- CHR tile viewer with live palette switching
- Pattern table grayscale visualizer
- Tile extractor for ROM analysis

#### GBA Save Files
- 5 distinct hardware types: None, EEPROM (512B/8KB), SRAM (32KB), Flash (64KB/128KB)
- Detection via embedded string markers (`EEPROM_V`, `SRAM_V`, `FLASH_V`, `FLASH1M_V`)
- EEPROM size ambiguity resolved by monitoring first DMA3 transfer (9 or 17 halfwords)
- **Critical detail:** EEPROM addressing in 64-bit BLOCKS, not bytes
- Flash RAM manufacturer ID variations (different protocols per chip)

**Tool opportunities:**
- GBA save parser/converter
- Auto-detector with game database fallback
- EEPROM protocol analyzer

#### Procedural Low-Poly Terrain (arXiv:2505.09350)
- Academic paper on terraced terrain generation (May 2025)
- Problem: Grid-based heightmaps too smooth/regular for low-poly aesthetic
- Solution: Height quantization into discrete terrace levels + biome regions
- Relevant to: `psx-terra`, `gba-terra`, `ascii-terrain`, `t3rra1n`

**Pipeline concept:**
1. Base noise (Perlin/Simplex)
2. Quantize heights → discrete levels
3. Marching squares for contour polygons
4. Biome assignment (moisture + temp maps)
5. PSX post-processing (vertex jitter, color quantization, dithering)

**Technical highlights:**

**iNES format parser:**
```python
Header: 16 bytes
- Magic: 'NES\x1a' (4 bytes)
- PRG-ROM banks: 16KB units
- CHR-ROM banks: 8KB units (512 tiles per bank)
- Mapper, mirroring, battery-backed flags
```

**2-bitplane tile decoder:**
```python
# 8×8 pixels = 16 bytes (2 bitplanes)
for y in range(8):
    plane0 = tile_bytes[y]      # Low bit
    plane1 = tile_bytes[y + 8]  # High bit
    for x in range(8):
        bit0 = (plane0 >> (7 - x)) & 1
        bit1 = (plane1 >> (7 - x)) & 1
        color_idx = (bit1 << 1) | bit0  # 0-3
```

**Half-block rendering:**
- Terminal cell: 1 char wide, 1 line tall
- Half-block (▀): Top half colored (fg), bottom half colored (bg)
- 8×8 tile → 2×4 terminal grid (2 chars wide, 4 lines tall)
- **Advantage:** 2:1 vertical resolution vs full blocks (8 pixels in 4 lines)
