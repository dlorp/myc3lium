### WebGL Rendering Failure (Canvas Black Screen)

**Spawned:** agent:strategic-planning-architect:subagent:d4021cb5-da7a-452f-a945-9b960ef96660  
**Delivered:** `~/repos/myc3lium/plans/canvas-rendering-fix-2026-03-22.md`

**Root causes identified:**

1. **Font atlas race condition (PRIMARY):**
   - Draw `useEffect` runs before font atlas loads async
   - `fontAtlasRef.current` not in dependency array
   - When atlas loads, draw never re-triggers → canvas stays black forever

2. **WebGL texture dimension mismatch:**
   - Error: `GL_INVALID_VALUE: glCopyTexSubImage2D: Offset overflows texture dimensions`
   - Phosphor render target sized to content (640×400) but framebuffer is viewport-sized
   - Even with effects disabled, error persists

3. **Duplicate render target useEffect:**
   - Two identical blocks create/dispose same ref
   - Potential use-after-dispose

**Fix #1 Applied (Atlas Load State):**
```jsx
const [atlasLoaded, setAtlasLoaded] = useState(false)

// In atlas load callback:
setAtlasLoaded(true)

// In draw useEffect deps:
}, [content, charHeight, charWidth, resolution, texture, contentError, textureLoadError, atlasLoaded])
```

**Commit:** `2e20e22` - "fix: add atlasLoaded state to trigger re-draw when font loads"

**Testing:**
- Mac localhost:3000 — Still black
- Pi 192.168.40.19:3000 — Not tested (deployment aborted due to Mac failure)

**Status:** Fix #1 insufficient. WebGL rendering fundamentally broken.
