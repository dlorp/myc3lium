**Deliverable:** v0.1 production release (commit: 727e144)

**Files created (6 total, 33 KB):**
- `nes_decoder.py` (8.3 KB) — iNES parser + 2-bitplane tile decoder + NES palette (64 colors) + 8 presets
- `nes_pattern_tui.py` (10.5 KB) — Textual TUI with PatternTableView + PaletteBar + HelpScreen
- `README.md` (4.8 KB) — Technical details, use cases, keybinds
- `DESIGN.md` (9.5 KB) — Philosophy, half-block rendering, palette semantics
- `requirements.txt`, `.gitignore`

**Features:**
- **Dual pattern tables:** Side-by-side view (bank 0 + bank 1, 512 tiles total)
- **8 palette presets:** Grayscale, Mario, Luigi, Sky, Fire, Ice, Gold, Purple
- **Live palette switching:** Press 1-8 → instant change (no mode, no confirmation)
- **Half-block rendering:** Unicode ▀ for 2:1 vertical resolution (8 pixels in 4 terminal lines)
- **Vim navigation:** j/k, h/l (switch banks), g/G, PgUp/PgDn
- **PSX aesthetic:** Dark blue BG, cyan/magenta borders, clean typography
- **Help toggle:** Press ? → show keybinds (hidden by default)

**Technical highlights:**
- iNES format parser (PRG-ROM + CHR-ROM extraction)
- 2-bitplane decoder: `color_idx = (bit1 << 1) | bit0` (0-3 range)
- NES NTSC palette: 64 colors RGB (tuples)
- Half-block: Top pixel = fg color, bottom pixel = bg color (2 pixels per terminal line)

**Testing:** Decoder unit tests (iNES parsing, tile decoding, palette application), demo ROM generator included

**Key insight from research:**
> NES tiles store **SHAPE** (CHR-ROM), not **COLOR** (palette at runtime).  
> Same tile + different palette = different sprite (e.g., Mario vs Luigi share tile data).

**Use cases:**
- ROM hackers: Preview sprite edits with different palettes
- Researchers: Analyze tile reuse patterns across games
- Artists: Study NES color constraint techniques (4 colors per sprite)
- Educators: Demonstrate 2-bitplane tile encoding (bitwise operations)
